User guide¶

Configuring the GitHub session¶

All the configuration of how to use the GitHub API v3 is done by calling several methods on a Builder object and finally calling its Builder.Build() method.

For example, you choose the authentication type by calling Builder.OAuth() or Builder.Login().

A complete example:

>>> import PyGithub

>>> b = PyGithub.BlockingBuilder()
>>> b.Login("your_login", "your_password")  
<...Builder object at 0x...>
>>> b.UserAgent("MyWonderfulApplication-1.0.0")  
<...Builder object at 0x...>
>>> g = b.Build()

Because all methods return the builder itself, you can chain calls:

>>> g = PyGithub.BlockingBuilder().Login("your_login", "your_password").UserAgent("MyWonderfulApplication-1.0.0").Build()

For reference, see Builder.

Common functionalities¶

Access to the session¶

All objects returned by PyGithub have a Session attribute. You can use it to access session-wide information, like your rate limit in RateLimit.

Conditional requests¶

Many objects returned by PyGithub have a update() method. You can use it to make a conditional request. The object’s attributes will be updated, and your rate limit consumed, only if the object has changed since it was created or last updated.

Pagination¶

Pagination is implemented in PyGithub by PaginatedList. You should use this class as a classic Python iterator type.

Example:

>>> user = g.get_authenticated_user()
>>> repositories = user.get_repos()  # repositories is a PaginatedList
>>> for repo in repositories:
...     print repo.full_name  
jacquev6/ActionTree
jacquev6/AnotherPyGraphvizAgain
...
jacquev6/ViDE
jacquev6/vincent-jacques.net

or shorter:

>>> for repo in user.get_repos():
...     print repo.full_name  
jacquev6/ActionTree
jacquev6/AnotherPyGraphvizAgain
...
jacquev6/ViDE
jacquev6/vincent-jacques.net

You can also build a list from a PaginatedList (all the subsequent requests to GitHub will be performed during the conversion instead of lazilly during the iteration):

>>> repositories = list(user.get_repos())  # repositories is a plain list

GitHub provide no way to know the number of items a paginated request will return, so PaginatedList has no length:

>>> len(user.get_repos())
Traceback (most recent call last):
...
TypeError: object of type 'PaginatedList' has no len()

If you really mean to take the length of a PaginatedList, you have to explicitelly construct a list and then use its length:

>>> len(list(user.get_repos()))
25

But often, there is an attribute in the parent object:

>>> user.public_repos
21
>>> user.owned_private_repos
4

For reference, see PaginatedList.

Naming conventions¶

Methods and attributes mapping exactly to the GitHub API v3 are in lower case like AuthenticatedUser.create_repo() and Repository.stargazers_count.

Methods and attributes more specific to PyGithub are in upper camel case like Session.RateLimit.

Access to common resources¶

Users, organizations¶

You can access a user by Github.get_user() and the authenticated user by Github.get_authenticated_user(). For orgs, use Github.get_org():

>>> nvie = g.get_user("nvie")
>>> print nvie.name
Vincent Driessen

>>> me = g.get_authenticated_user()
>>> print me.login
jacquev6
>>> print me.name
Vincent Jacques

>>> github = g.get_org("github")
>>> print github.location
San Francisco, CA

Repositories¶

There are several ways to access a repository:

  • if you already have the User, AuthenticatedUser or Organization owning the repo, call its get_repo() method:

    >>> gitflow = nvie.get_repo("gitflow")
>>> print gitflow.description
Git extensions to provide high-level repository operations for Vincent Driessen's branching model.

    >>> pygithub = me.get_repo("PyGithub")
>>> print pygithub.description
Python library implementing the full GitHub API v3

  • if you have its full name, use Github.get_repo():

    >>> whisper = g.get_repo("graphite-project/whisper")  # You don't have to know if graphite-project is a user or an org
>>> whisper.created_at
datetime.datetime(2012, 5, 7, 21, 30, 26)

  • if you have the login of its owner and its name, you can call Github.get_repo() with a tuple:

    >>> apidoc = g.get_repo(("github", "developer.github.com"))
>>> print apidoc.description
GitHub Developer site

  • if you want to list someone’s repositories, use get_repos():

    >>> for repo in g.get_org("graphite-project").get_repos():
...     print repo.name
carbon
whisper
graphite-web
ceres
graphite-project.github.io
